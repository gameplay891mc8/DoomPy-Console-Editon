# 😈 DOOM: CONSOLE EDITION // G Studio-Team 😈

**DOOM: Console Edition** — это уникальный текстовый 2D-шутер, созданный специально для запуска внутри консоли терминала. Проект прошел путь от пошагового текстового тира до динамичного экшена в реальном времени!

> **Разработчик и основатель:** gameplay891mc8 // [G Studio-Team](https://github.com) 🚀

## 📅 История версий проекта (Changelog)

### 🚀 ЭПОХА REBOOT: Версия v0.0.1 (Текущая)
Полный перезапуск проекта со свежей головы с переходом на движок реального времени!
* **Движок Curses:** Полный отказ от бесконечных пауз и необходимости постоянно нажимать `Enter` после каждого действия. Игра считывает клавиши мгновенно на лету!
* **Реальное время (ИИ монстров):** Демоны больше не ждут твоего хода. Они делают шаг вперед каждые 1.2 секунды. Замешкался — тебя съели.
* **Интерактивное меню:** Выбор языка (Русский / English) прямо в терминале с помощью стрелочек на клавиатуре.
* **Динамический фидбек:** Добавлена визуальная вспышка выстрела дробовика (`💥💥💥` окрашивается в красный цвет на долю секунды) и системный звуковой сигнал (Beep).

### ⏳ СТАРАЯ ВЕРСИЯ: Classic Console Edition (Архив)
Первая историческая сборка игры, с которой все начиналось.
* **Пошаговый геймплей:** Игра двигалась по принципу текстовой стратегии через стандартный ввод `input()`. Монстры делали шаг только после того, как игрок отправлял команду через `Enter`.
* **Текстовые команды:** Для каждого действия нужно было прописывать буквы в консоль (например, `f` для огня или `r` для перезарядки).
* **База кода:** Простой линейный цикл, оптимизированный под стабильную работу на смартфонах Poco X6 Pro с HyperOS без задействования тяжелых графических библиотек.

## 🕹️ Управление (Актуально для версии Reboot-0.0.1):
* `Стрелочки Вверх/Вниз` — Навигация в меню выбора языка.
* `Enter` — Подтверждение выбора языка.
* `F` / `f` — Огонь из дробовика (Fire).
* `R` / `r` — Перезарядка дробовика (+5 патронов).
* `Q` / `q` — Быстрый выход из игры в терминал.

## ⚠️ ВАЖНО: ИНСТРУКЦИЯ ПО ЗАПУСКУ REBOOT-ВЕРСИИ ⚠️

### 📱 На Android (в приложении PyDroid 3):
Если вы просто нажмете желтую кнопку **«Play»** внутри PyDroid 3, программа вылетит с ошибкой. Графический экран приложения не поддерживает интерактивные терминальные библиотеки.

**Запускайте игру строго по инструкции:**
1. Откройте боковое меню PyDroid 3 (три полоски в левом верхнем углу).
2. Выберите пункт **Terminal** (откроется настоящая черная консоль Linux).
3. Введите команду для запуска вашего файла: `python main.py` (или имя вашего файла).
4. Нажмите **Enter** на виртуальной клавиатуре телефона. Игра развернется на весь экран терминала!

### 🐧 На Linux (Linux Master Build):
В операционных системах Linux игра работает идеально «из коробки» в любом стандартном системном терминале.
```bash
python3 main.py
```

### 💻 На Windows (Для тестов):
Для работы библиотеки управления экраном на Windows необходимо один раз доустановить порт:
```cmd
pip install windows-curses
python main.py
```
